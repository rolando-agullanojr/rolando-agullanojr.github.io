Apache Cordova

 

 

For this lab, I will be talking about the Apache Cordova, created by the Apache Foundation, Version 2.0. This tool can be used on a variety of platforms, including Android, iOS, Windows, OS X, Electron. This tool is an open-source mobile development framework that is primarily used for cross-platform development. The main advantages of Apache Cordova are that it can extend across multiple platforms without having to re-implement the platform language. This also counts as the disadvantage as with most cross-platform frameworks are limited to the packaged defined itself, limiting the possible technologies offered in native platforms.

 

You can find the tool at: https://cordova.apache.org/

 

The software is developed by the Apache Foundation currently, but originally created by Nitobi, a development company. Apache Cordova had its first stable release in August 4, 2020 by the Apache Foundation, which explains the licensing of Apache License 2.0. Apache Cordova is an open-source software meaning that it is free and available to any that download the framework through the site stated about. The main languages defined in the framework are HTML, JavaScript, and CSS. The framework includes functionality, but not limited to:

-          Battery Status

-          Camera

-          Geolocation

-          In-app Browser

-          Media Capture

-          Screen Orientation

-          Splash screen

-          Status bar

-          Vibration

-          Whitelist

Apache Cordova also has different debugging apps, ranging from iOS Debugging applications like XCode or Safari Remote Debugging, to Chrome Remote Debugging. Cordova also contains a command-line interface and Intel-based emulator, allowing for an online CLI for development and emulation. The tool does not have hardware specific requirements but does require at least Windows 8.1 operating system. Apache Cordova does not have a built in IDE (Integrated Development Tool), but it is compatible with later versions of Microsoft Visual Studio. Cordova does not explicitly state mixed language support, but it does state English support for the framework. Cordova also does not explicitly state licensing for final product deployment, which is also makes sense for an open-source software. Third-party emulators and software can be added, in which the Cordova website states a variety of plug-ins to be added to the program, with extensive emulators.

In terms of business, the market share is not large, most likely from other competitors like Android Studio. Cordova takes up 6.34 percent of apps on a combination of markets (iOS, Android, etc). Cordova apps account for 4.21 percent of new applications being created, therefore the amount of applications created and supported under the framework is incredibly low. The installation rate is even lower for these applications, coming in at 1.09 percent.

On a brighter note, Apache Cordova is seen to be very popular for new developers that do not with to use Java language or Android Studio. Cordova also supports an invisible browser that keep their content separate from hardware of the mobile device, ensuring safety in terms of security for the device. Cordova adds a competitor in the market, in a couple of years we will see how successful it can become.
